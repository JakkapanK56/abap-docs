This example demonstrates the addition `REDIRECTED` of [`WITH ASSOCIATIONS`](ABAPWITH_ASSOCIATIONS.html) where the redirect is made to the current CTE.

As in the example [Exposing Association with a Redirect](ABENWITH_ASSOCS_REDIRECT_ABEXA.html), a common table expression `+cte` accesses the CDS view entity `DEMO_CDS_EXPOSE_ASSOC_1` and exposes its CDS association `_demo_join2`. Here, however, the redirect is made to the current common table expression and not to a preceding common table expression. This expression specifies the [association target](ABENASSOCIATION_TARGET_GLOSRY.html)\\ `demo_join2` of the exposed CDS association twice as an additional data source under the names `src1` and `src2`.

The results of the two reads are different from each other. The following reads again demonstrate which join instances are created. The different results come from the different `ON` conditions, where the column of `src1` and the column of `src2` are both evaluated once.

-   In the first `WITH` statement, the data source is specified using `src1`.
-   In the second `WITH` statement, the data source is specified using `src2`.
