`managed by BOPF [...];`

Defines a [BOPF-based RAP BO](ABENRAP_BOPF_BO_GLOSRY.html) in [BDL source code](ABENBDL_SOURCE_CODE_GLOSRY.html) in the [RAP BDL](ABENCDS_BDL_GLOSRY.html). A BOPF-based RAP BO is the result of migrating a [CDS-based BOPF business object](ABENCDS_BOPF_BO_GLOSRY.html) to a RAP business object. BOPF-based RAP BOs cannot be built from scratch.

The [transactional buffer](ABENTRANSACTIONAL_BUFFER_GLOSRY.html) of a BOPF-based RAP BO is provided by the [BOPF-managed RAP BO provider](ABENBOPF_MAN_RAP_BO_PROV_GLOSRY.html).

A BOPF-managed RAP BO has a limited feature set compared to a managed or unmanaged RAP BO. Since transactional requests are managed by the [BOPF-managed RAP BO provider](ABENBOPF_MAN_RAP_BO_PROV_GLOSRY.html), some features work differently than in a RAP-native RAP BO. For details, see the [Migration Guide for CDS-based BOPF Business Objects](https://help.sap.com/docs/ABAP_PLATFORM_NEW/0a54d0c8a2be4136a8b5d41a367dd537/40535a6f276149bfb1fd03baadd5ba92).

The following [RAP behavior definition](ABENCDS_BEHAVIOR_DEFINITION_GLOSRY.html)\\ `DEMO_RAP_BOPF` represents a BOPF-based RAP BO.

//GENERATED BY MIGRATION TOOL\\ 
managed by BOPF \\n  implementation in class BP\_DEMO\_RAP\_BOPF unique; \\nstrict(2); \\ndefine behavior for DEMO\_RAP\_BOPF \\npersistent table demo\_dbtab\_root \\nauthorization master ( instance, global ) \\nlock master \\n\\{ \\n  create; \\n  update; \\n  delete; \\n\\}
