This section covers [dialog processing](ABENDIALOG_PROCESSING_GLOSRY.html) in [dialog sessions](ABENDIALOG_SESSION_GLOSRY.html) based on [SAP GUI](ABENSAP_GUI_GLOSRY.html). The user dialogs are all based on classic [dynpros](ABENDYNPRO_GLOSRY.html). Both general and special dynpros (selection screens, lists) are possible. Dynpros can be linked with [GUI controls](ABENGUI_CONTROL_GLOSRY.html).

Dynpros can be used in function pools, module pools, and executable programs.

-   [User Dialogs](ABENSCREEN_ABEXA.html)
-   [General Dynpros](ABENABAP_DYNPROS.html)
-   [Selection Screens](ABENSELECTION_SCREEN.html)
-   [Classic Lists](ABENABAP_DYNPRO_LIST.html)
-   [Conversion Exits](ABENCONVERSION_EXITS.html)
-   [Accessing the Presentation Server](ABENFRONTEND_SERVICES.html)
-   [Example Output with `CL_DEMO_OUTPUT`](ABENCL_DEMO_OUTPUT.html)
-   [SAP GUI User Dialogs Glossary](ABENSAP_GUI_GLOSSARY.html)

-   The dialog processing shown here are closely linked with the ABAP programming language, which means that the handling of user dialogs is performed mainly by ABAP language elements.
-   When programming user dialogs, it is important that the presentation logic and application logic are decoupled.
-   For Web-based UIs, [SAPUI5](ABENSAPUI5_GLOSRY.html), [Web Dynpros](ABENWEB_DYNPRO_GLOSRY.html) and [Business Server Pages](ABENBUSINESS_SERVER_PAGES_GLOSRY.html) are available, which are independent of the classic dynpros and are addressed using object-oriented frameworks.
