[ABAP operands](ABENABAP_OPERAND_GLOSRY.html) whose values can be evaluated in [read positions](ABENREAD_POSITION_GLOSRY.html) or modified in [write positions](ABENWRITE_POSITION_GLOSRY.html) are:

The following topics provide a detailed description of data objects and expressions as operands in [operand positions](ABENOPERAND_POSITION_GLOSRY.html).

Besides data objects and expression, there are also other kinds of operands that are specified in other operand positions than read and write positions. Examples of those are:

-   [Elementary operands](ABENELEMENTARY_OPERAND_GLOSRY.html)
-   Elementary operands are [data objects](ABENDATA_OBJECT_GLOSRY.html) that can be specified in various ways, either directly or using [field symbols](ABENFIELD_SYMBOL_GLOSRY.html) or [reference variables](ABENREFERENCE_VARIABLE_GLOSRY.html) and so on.
-   [Expressions](ABENEXPRESSION_GLOSRY.html)
-   At many read positions and some write positions, expressions can be specified instead of data objects. The statement then works with the result of the expression.

-   [Data Objects in Operand Positions](ABENOPERANDS_DATA_OBJECTS.html)
-   [Expressions in Operand Positions](ABENOPERANDS_EXPRESSIONS.html)

-   Type specifications
-   Statements can refer to types such as data types, classes, or interfaces. Examples are [declarations](ABENABAP_DECLARATIONS.html), [typings](ABENTYPING.html), or when [addressing components](ABENOPERANDS_NAMES.html).
-   Callable units from the same program or the repository
-   Statements can call units such as [procedures](ABENPROCEDURE_GLOSRY.html) or ABAP programs. See [Calling and Exiting Program Units](ABENABAP_EXECUTION.html).
-   Resources
-   Statements can access resources such as CDS entities, DDIC database or other resources. Examples are the statements of [ABAP SQL](ABENABAP_SQL_GLOSRY.html).
