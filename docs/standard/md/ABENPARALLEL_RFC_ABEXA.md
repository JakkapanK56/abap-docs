This example demonstrates parallel asynchronous RFC processing.

Parallel asynchronous processing of the function module `RFC_SYSTEM_INFO` using [asynchronous remote function calls](ABENASYNCHRONOUS_RFC_GLOSRY.html). Ten calls with different task IDs `name` are made, which each run in a separate [work process](ABENWORK_PROCESS_GLOSRY.html). The callback routine `callbback_meth` counts the completed function modules and receives information about the target system.

The addition `GROUP DEFAULT` is used to distribute the execution across all [AS instances](ABENAS_INSTANCE_GLOSRY.html) of the current AS ABAP. If no more free work processes are available after at least one successful call, the execution is stopped until all function modules started up to that point have been completed. This interruption is limited to a maximum of 5 seconds.

After all function modules have been started, the system waits until all callback routines have been executed. After that, the internal table `task_list` filled there is produced. The output shows the order in which the individual tasks were completed and on which AS instance they were executed.
