This example demonstrates how [global temporary tables](ABENDDIC_DATABASE_TABLES_GTT.html) in ABAP Dictionary are accessed.

[ABAP SQL](ABENABAP_SQL_GLOSRY.html) and [Native SQL](ABENNATIVE_SQL_GLOSRY.html) are used to access a [global temporary table (GTT)](ABENGLOBAL_TEMPORARY_TABLE_GLOSRY.html)\\ `DEMO_GTT`:

This example uses Native SQL to fill the GTT for demonstration purposes only. Only ABAP SQL should be used to access GTTs.

-   After the GTT is filled using the ABAP SQL statement [`INSERT`](ABAPINSERT_DBTAB.html), the table is cleared again using the ABAP SQL statement [`DELETE FROM`](ABAPDELETE_DBTAB.html) without a `WHERE` condition. If the `DELETE` statement is not executed, the statement [`WAIT UP TO`](ABAPWAIT_UP_TO.html), which performs an implicit [database commit](ABENDATABASE_COMMIT_GLOSRY.html), produces the runtime error `COMMIT_GTT_ERROR`, which can be found in the ABAP dump analysis (`ST22`).
-   After the GTT has been filled using a static native `INSERT` statement between [`EXEC SQL - ENDEXEC`](ABAPEXEC.html), the table is not cleared explicitly. Instead, it is cleared in most database platforms using an implicit [database commit](ABENDATABASE_COMMIT_GLOSRY.html) raised by the statement [`WAIT UP TO`](ABAPWAIT_UP_TO.html).
