Content of [reference variables](ABENREFERENCE_VARIABLE_GLOSRY.html). A valid reference refers to a [data object](ABENDATA_OBJECT_GLOSRY.html) ([data reference](ABENDATA_REFERENCE_GLOSRY.html)) or an [instance](ABENINSTANCE_GLOSRY.html) of a [class](ABENCLASS_GLOSRY.html) ([object reference](ABENOBJECT_REFERENCE_GLOSRY.html)). References in reference variables are either [heap references](ABENHEAP_REFERENCE_GLOSRY.html) or [stack references](ABENSTACK_REFERENCE_GLOSRY.html). Internally, [strings](ABENSTRING_GLOSRY.html), [internal tables](ABENINTERNAL_TABLE_GLOSRY.html), and [boxed components](ABENBOXED_COMPONENT_GLOSRY.html) are also addressed via references. From a technical point of view, an object in ABAP is referenced in multiple layers. A reference is the address of an internal header (object header, string header, or table header), which contains the address of the actual object and other administrative information such as the [data type](ABENDATA_TYPE_GLOSRY.html) of a referenced data object.
