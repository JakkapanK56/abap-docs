This example demonstrates valid literal values for all [built-in dictionary types](ABENDDIC_BUILTIN_TYPES.html).

The first part of the example uses the ABAP SQL statement [`INSERT`](ABAPINSERT_DBTAB.html) to insert a value into each column of a row of the DDIC database table `DEMO_DDIC_TYPES`. The values are [ABAP literals](ABENABAP_SQL_LITERALS.html) used in a [host expression](ABENABAP_SQL_HOST_EXPRESSIONS.html). It should be noted that the literals are assigned to the components of a temporary ABAP structure that is constructed with the [`VALUE`](ABENCONSTRUCTOR_EXPRESSION_VALUE.html) operator. These components have the ABAP type that is mapped to the DDIC type of the respective database column. If the type of a literal does not match the this ABAP type, its value is [converted according to the rules](ABENCONVERSION_ELEMENTARY.html). When the temporary structure is inserted in the database table via the [database interface](ABENDATABASE_INTERFACE_GLOSRY.html), another conversion might take place from the ABAP types to the database specific types, if necessary.

The second part of the example uses the ABAP SQL statement [`UPDATE`](ABAPINSERT_DBTAB.html) to set the values of the columns of a row that was inserted with empty columns. In the `UPDATE` statement, the values for most of the data types can be passed as [typed literals](ABENABAP_SQL_TYPED_LITERALS.html) that are not possible in general ABAP. In the case of typed literals, no ABAP-specific type conversion takes place and the validity of the values is guaranteed by the syntax check.
