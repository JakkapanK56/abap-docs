Demonstrates access to tables in explicitly specified [database schemas](ABENDATABASE_SCHEMA_GLOSRY.html). The AMDP methods do not use any AMDP options for [client handling](ABAPMETHODS_AMDP_OPTIONS_CLIENT.html) and are not [client-safe](ABENCLIENT_SAFE_GLOSRY.html).

This example accesses [AMDP methods](ABENAMDP_METHOD_GLOSRY.html) that are declared and implemented in the [AMDP class](ABENAMDP_CLASS_GLOSRY.html)\\ `CL_DEMO_AMDP_DB_SCHEMA`.

-   The method `GET_SCHEMAS_PHYSICAL` accesses the table `SCHEMAS` of the physical database schema `SYS` by specifying the schema directly. All existing database schema are stored in this table.

-   The method `GET_SCHEMAS_LOGICAL` accesses the physical database schema that is mapped to the [logical local database schema](ABENLOGICAL_DATABASE_SCHEMA_GLOSRY.html)\\ `DEMO_LOGICAL_DB_SCHEMA` in transaction `DB_SCHEMA_MAP` by specifying the [AMDP macro](ABENAMDP_MACRO_GLOSRY.html)\\ [`$ABAP.schema`](ABENAMDP_LOGICAL_DB_SCHEMAS.html). If this is `SYS`, the same table is accessed.

-   Another method `GET_SCHEMAS_LOGICAL_TO_ABAP` demonstrates how the addition [`USING SCHEMA`](ABAPMETHOD_BY_DB_PROC.html) of the statement `METHOD` is specified. The [logical local database schema](ABENLOGICAL_DATABASE_SCHEMA_GLOSRY.html)\\ `DEMO_LOGICAL_DB_SCHEMA_TO_ABAP` is defined in such a way that it allows access to the [ABAP database schema](ABENABAP_DB_SCHEMA_GLOSRY.html). Since it is used in the method in the macro [`$ABAP.schema`](ABENAMDP_LOGICAL_DB_SCHEMAS.html), it must be declared after `USING SCHEMA`. The access to database table `SCARR` is cross-client.

-   If the current ABAP database schema is mapped to the logical local database schema using the predefined name `:abap_db_schema`, the full string `"$ABAP.schema( DEMO_LOGICAL_DB_SCHEMA_TO_ABAP )".` is omitted when the macro is evaluated and the ABAP database schema is accessed implicitly. The specification of `scarr` is checked statically against the ABAP Dictionary.
-   If another physical database schema is mapped to the logical local database schema, there must be a suitable database object `SCARR` in this database schema when the method is executed.
