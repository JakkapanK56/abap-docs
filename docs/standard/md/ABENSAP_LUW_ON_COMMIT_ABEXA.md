This example demonstrates how SAP LUWs are bundled using subroutines.

This example works mainly in the same way as the [executable example](ABENSAP_LUW_UPDATE_TASK_ABEXA.html) for [updates](ABENUPDATE_GLOSRY.html), but demonstrates bundling using subroutines registered using the statement [`PERFORM ON COMMIT`](ABAPPERFORM_ON_COMMIT.html).

No parameters can be passed in this type of bundling, which means that the internal table used in the database writes is passed by the statements [`EXPORT`](ABAPEXPORT_DATA_CLUSTER.html) and [`IMPORT`](ABAPIMPORT_DATA_CLUSTER.html) to and from the [ABAP Memory](ABENABAP_MEMORY_GLOSRY.html).

The registered subroutines are called using the statements [`COMMIT WORK`](ABAPCOMMIT.html), which can be monitored using the ABAP Debugger. The subroutines themselves do not implement the writes itself but call methods of a class instead as recommended.
