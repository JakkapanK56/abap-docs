This example demonstrates how an SQLScript procedure is implemented using AMDP. Since it implements a write access and is not [client-safe](ABENCLIENT_SAFE_GLOSRY.html), the AMDP class cannot have the ABAP language version [ABAP for Cloud Development](ABENABAP_FOR_CLOUD_DEV_GLOSRY.html).

A simple [SQLScript procedure](ABENSQL_SCRIPT_PROCEDURE_GLOSRY.html) is implemented in three AMDP methods, showing different options for user-defined [client handling](ABENAMDP_CLIENT_HANDLING.html), in the AMDP class `CL_DEMO_AMDP`:

The database table `SPFLI` defined in the ABAP Dictionary must be specified after `USING`. On an SAP HANA database, this class works in the same way as the executable examples for [ADBC](ABENADBC_PROCEDURE_ABEXA.html) and for [`CALL DATABASE PROCEDURE`](ABENCALL_DB_PROCEDURE_ABEXA.html). AMDP replaces these technologies when calling database procedures for an SAP HANA database that is the [standard database](ABENSTANDARD_DB_GLOSRY.html) of an AS ABAP.

This is a syntax example. The same function can be achieved with the same efficiency in ABAP SQL. AMDP is not needed in simple cases like this.

-   `INCREASE_PRICE`

-   `INCREASE_PRICE_CLIENT`

-   `INCREASE_PRICE_CDS_CLIENT`
