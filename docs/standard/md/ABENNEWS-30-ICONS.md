From ABAP release 3.0, icons can be represented on [dynpros](ABENDYNPRO_GLOSRY.html). Either Screen Painter is used to define the icons or the function module `ICON_CREATE` for setting icons dynamically from an ABAP program. <`ICON`\> must be included in the program so that the icons can be addressed using their symbolic names. It is important that the [dynpro field](ABENDYNPRO_FIELD_GLOSRY.html) that contains an icon at runtime is at least as long as the identically named ABAP field. Apart from the icon name, the field should also be able to accommodate a short information text.
