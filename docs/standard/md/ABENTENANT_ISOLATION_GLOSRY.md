Tenant isolation means that the data of one tenant is isolated from the other [tenants](ABENTENANT_GLOSRY.html). In a [multitenancy](ABENMEGATENANCY_GLOSRY.html) system, the [ABAP runtime framework](ABENABAP_RUNTIME_FRMWK_GLOSRY.html) supports tenant isolation by allowing an [application program](ABENAPPLICATION_PROGRAM_GLOSRY.html) to access only the data of the [application client](ABENAPPLICATION_CLIENT_GLOSRY.html) assigned to the current tenant. Read access to the [system client](ABENSYSTEM_CLIENT_GLOSRY.html) is also possible. [Multitenancy](ABENMEGATENANCY_GLOSRY.html) is not supported in the current release.
