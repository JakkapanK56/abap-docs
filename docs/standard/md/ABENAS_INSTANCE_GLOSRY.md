AS instance for short. Component of an [AS ABAP](ABENAS_ABAP_GLOSRY.html). An AS ABAP can be implemented using one or more AS instances, which are usually instantiated on different [host computers](ABENHOST_COMPUTER_GLOSRY.html). However, multiple AS instances of one or more AS ABAP can also be instantiated on the same host computer. Each AS instance has [work processes](ABENWORK_PROCESS_GLOSRY.html) to execute [ABAP programs](ABENABAP_PROGRAM_GLOSRY.html). The memory areas of an AS instance that can be accessed in the ABAP programs are the [shared memory](ABENSHARED_MEMORY_GLOSRY.html), the [user memory](ABENUSER_MEMORY_GLOSRY.html), the [ABAP memory](ABENABAP_MEMORY_GLOSRY.html), and the [session memory](ABENROLL_AREA_GLOSRY.html) in the [internal session](ABENINTERNAL_SESSION_GLOSRY.html).
