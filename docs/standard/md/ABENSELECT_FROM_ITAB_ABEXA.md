This example demonstrates how an internal table is accessed using an ABAP SQL [query](ABENASQL_QUERY_GLOSRY.html).

An internal table is filled with random values and is used as a [data source](ABAPSELECT_ITAB.html) of a [`SELECT`](ABAPSELECT.html) statement. Aggregate expressions, an SQL expression, `GROUP BY`, and `ORDER BY` are all used when doing this. The statement cannot be processed by the [ABAP SQL in-memory engine](ABENABAP_SQL_INMEMENG_GLOSRY.html) since it does not meet the corresponding [restrictions](ABENSQL_ENGINE_RESTR.html). The pragma `##itab_db_select` is used to hide the associated syntax check warning.
