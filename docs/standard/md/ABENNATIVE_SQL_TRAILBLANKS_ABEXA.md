This example demonstrates a difference between ABAP SQL and Native SQL regarding the handling of trailing blanks.

The columns `STRING` and `CHAR10` of DDIC database table `DEMO_DDIC_TYPES` is updated by using [set expressions](ABENSET_EXPRESSION_GLOSRY.html) in [ABAP SQL](ABENABAP_SQL_GLOSRY.html), [ADBC](ABENADBC_GLOSRY.html) and with [`EXEC SQL`](ABENNATIVESQL.html).

-   The column `STRING` has the built-in data type [`STRING`](ABENDDIC_BUILTIN_TYPES.html#abenddic-builtin-types-3--that-is-mapped-to-the-hana-type--nclob--------when-updating-a-column-of-ddic-type--string--from-a-text-field-of-type--c--in-abap-sql--the-trailing-blanks-are-truncated-according-to-the-abap-rules------when-updating-the-same-column-with-the-same-text-field-in-native-sql--adbc---exec-sql----no-truncation-occurs-because-of-the-hana-rules-for-type--nclob--------the-column--char10--has-the-built-in-data-type---char---abenddic-builtin-types-html-@ITOC@@ABENDDIC_BUILTIN_TYPES_3) with length of 10 that is mapped to the HANA type `NVARCHAR` with that length.

-   When updating a column of DDIC type `CHAR` from a [text field literal](ABENTEXT_FIELD_LITERAL_GLOSRY.html) in ABAP SQL, the trailing blanks are truncated according to the ABAP rules.
-   When updating the same column with the same text field literal in Native SQL (ADBC, `EXEC SQL`), [HANA SQL](https://help.sap.com/docs/SAP_HANA_PLATFORM/4fe29514fd584807ac9f2a04f6754767/b4b0eec1968f41a099c828a4a6c8ca0f) writes the trailing blanks into the column.

-   When trying to select the row with ABAP SQL using a `WHERE` condition that compares with a text string that contains the same data as written, only the data written with ABAP SQL are found, because the trailing blanks of the operand of the `WHERE` condition are not considered. In order to find all data, the function [`LEFT`](ABENSQL_STRING_FUNC.html) can be used.
