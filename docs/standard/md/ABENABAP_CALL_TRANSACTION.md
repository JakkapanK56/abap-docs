A transaction is the execution of a program using a [transaction code](ABENTRANSACTION_CODE_GLOSRY.html). The process flow of the program depends on which [initial dynpro](ABENINITIAL_DYNPRO_GLOSRY.html), [selection screen](ABENSELECTION_SCREEN_GLOSRY.html), or method the transaction code is linked to.

There are two statements for calling transactions:

-   [`CALL TRANSACTION`](ABAPCALL_TRANSACTION.html)
-   [`LEAVE TO TRANSACTION`](ABAPLEAVE_TO_TRANSACTION.html)
