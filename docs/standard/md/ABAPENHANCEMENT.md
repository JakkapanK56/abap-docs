[Short Reference](ABAPENHANCEMENT_SHORTREF.html)

`ENHANCEMENT id.`\\ 
  `...`\\ 
`ENDENHANCEMENT.`

These statements define a [source code plug-in](ABENSOURCE_CODE_PLUGIN_GLOSRY.html) with the ID `id`. A [source code enhancement](ABENSOURCE_CODE_ENHANCEMENT_GLOSRY.html) can be implemented between the statements. The [control structures](ABENCONTROL_STRUCTURE_GLOSRY.html) and [processing blocks](ABENPROCESSING_BLOCK_GLOSRY.html) used in the implementation must be complete.

The statements `ENHANCEMENT` and `ENDENHANCEMENT` cannot be entered and edited directly. Instead, they are generated by the [ABAP Workbench](ABENABAP_WORKBENCH_GLOSRY.html) when [enhancements](ABENENHANCEMENT_GLOSRY.html) are created with the [ABAP Enhancement Framework](ABENENHANCEMENT_FRAMEWORK_GLOSRY.html). The ID `id` is also assigned by the workbench.

-   In the terminology of the [ABAP Enhancement Framework](ABENENHANCEMENT_FRAMEWORK_GLOSRY.html), a source code plug-in is an [enhancement implementation element](ABENENHANCEMENT_IMPL_ELEM_GLOSRY.html) of a simple [enhancement implementation](ABENSIMPLE_ENHANCEMENT_IMPL_GLOSRY.html).
-   Although source code plug-ins appear in the same source code as the corresponding [enhancement options](ABENENHANCEMENT_POINT_GLOSRY.html)\\ [`ENHANCEMENT-POINT`](ABAPENHANCEMENT-POINT.html) or [`ENHANCEMENT-SECTION`](ABAPENHANCEMENT-SECTION.html) in the ABAP Editor, they are actually stored in other [include programs](ABENINCLUDE_PROGRAM_GLOSRY.html) that are managed by the [Enhancement Builder](ABENENHANCEMENT_BUILDER_GLOSRY.html).
-   It is possible to enhance a source code plug-in using further source code plug-ins. This means that, from then onwards, the statements [`ENHANCEMENT-POINT`](ABAPENHANCEMENT-POINT.html) and [`ENHANCEMENT-SECTION`](ABAPENHANCEMENT-SECTION.html) can be used between `ENHANCEMENT` and `ENDENHANCEMENT`.
