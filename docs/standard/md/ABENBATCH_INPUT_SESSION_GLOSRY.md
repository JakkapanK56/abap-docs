A summary of a series of transaction calls including input data and [user actions](ABENUSER_ACTION_GLOSRY.html). A batch input session can be used to execute [dialog transactions](ABENDIALOG_TRANSACTION_GLOSRY.html) in [batch input](ABENBATCH_INPUT_GLOSRY.html), whereby some or all of the [screens](ABENSCREEN_GLOSRY.html) are handled by the session. Batch input sessions are stored persistently and are managed by the batch input administration in transaction `SM35`. See also [batch input tables](ABENBATCH_INPUT_TABLE_GLOSRY.html).
