This example demonstrates how a [database procedure](ABENDATABASE_PROCEDURE_GLOSRY.html) created in a class is called.

[ADBC](ABENADBC_GLOSRY.html) is used in the method `create_db_procedure` to create the same database procedure as in the executable example [ADBC, Stored Procedure](ABENADBC_PROCEDURE_ABEXA.html). Here, however, the procedure is called using the statement [`CALL DATABASE PROCEDURE`](ABAPCALL_DATABASE_PROCEDURE.html) and not using ADBC.

The [database procedure proxy](ABENDATABASE_PROC_PROXY_GLOSRY.html) required for this is created temporarily in the class itself using the associated API.

The executable example [AMDP, simple procedure call](ABENAMDP_ABEXA.html) shows how the procedure can be managed and called as an [ABAP Managed Database Procedure](ABENAMDP.html).

-   The API is created using the factory method `GET_PROXY_PUBLIC_API` from the factory class `CL_DBPROC_PROXY_FACTORY`.
-   The proxy is created using the method `CREATE_PROXY` of the interface `IF_DBPROC_PROXY_PUBLIC_API`.
-   Here, the internal table `params` filled previously is used to define the mapping between the parameter interface and ABAP data types. The parameter names are also modified.
