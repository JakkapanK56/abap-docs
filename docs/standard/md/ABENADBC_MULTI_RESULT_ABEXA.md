This example demonstrates how to access multiple result sets with ADBC.

The example creates a database procedure `ABAP_DOCU_DEMO_TWO_RESULTS` on a [SAP HANA database](ABENHANA_DATABASE_GLOSRY.html) that has two tabular output parameters. The parameters are filled with different values by a built-in SQL function. When calling the procedure with method `EXECUTE` of ADBC class `CL_SQL_STATEMENT`, the return value references an object of class `CL_SQL_RESULT_SET` that allows access to both result sets. After the first result set is accessed, the method `NEXT_RESULT_SET` is used to switch to the next one. An ABAP data object must be assigned to each result set to read the result. Here, internal tables are assigned with `SET_PARAM_TABLE`.
