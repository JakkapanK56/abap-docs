If the program property [*fixed point arithmetic*](ABENFIXED_POINT_ARITHMETIC_GLOSRY.html) is not set, the decimal separator in source fields with the type [`p`](ABENBUILTIN_TYPES_NUMERIC.html) is ignored, except in assignments to character-like target fields with the types `c` and `string`.

The result of the following conversion has the value `8.1499999999999995E-01`.

If the number of decimal places in the source field is greater than the number of digits calculated from 2 x the length of `dobj` - 1, the corresponding number of zeros is inserted between decimal separators and digits in assignments to data objects of the types `c` and `string` and the longer string of digits is assigned.

The string resulting from the conversion is `123.456-`.

The byte chain resulting from the conversion is `0000007C` and matches the result of the conversion of the rounded number `124`.

The method `TSTMP2UTCLONG` of the class [`CL_ABAP_TSTMP`](ABENCL_ABAP_TSTMP.html) converts UTC [time stamps in packed numbers](ABENTIME_STAMPS_PACKED.html) to [time stamp fields](ABENTIMESTAMP_FIELD_GLOSRY.html) of the type `utclong`.

The value of the date field resulting from the conversion is `20170111`.

If the number of places for a target field of type `decfloat16` is greater than 16 when the assignment is performed, commercial rounding to 16 places is applied. If the mantissa of the target field is long enough, the [scaling](ABENSCALE_GLOSRY.html) is set to the number of [decimal places](ABENDECIMAL_PLACE_GLOSRY.html) of the source field. An invalid value in the source field raises the catchable exception `CX_SY_CONVERSION_NO_NUMBER`. `f` The value of the packed number is converted into the internal format of a binary floating point number. If the decimal number cannot be represented as a binary floating point number, the nearest value is used. An invalid value in the source field produces undefined behavior. `c` The value of the packed number is formatted in [commercial notation](ABENNUMERICAL_VALUE.html) and passed right-aligned to the target field. The character *\-* is set in the last position for a negative value and a blank is set in the last place for a positive value. If the target field is longer than the string of digits, including the plus/minus sign, it is padded with blanks on the left. If it is too short, the number representation is moved to the right by one position for positive values. If the target field is still too short (and for negative values), it is truncated on the left and the character *\** is set in the first position of the target field. `n` The value of the packed number is rounded commercially to an integer number. The absolute value is passed to the target field as a right-aligned string of digits. If the target field is longer than the string of digits, the field is padded with zeros on the left. If it is too short, the values on the left are cut off. `string` The value of the packed number is formatted in [commercial notation](ABENCOMMERCIAL_NOTATION_GLOSRY.html) and passed to the target field without gaps. The character *\-* is set in the last position for a negative value and a blank is set in the last position for a positive value. The resulting length of the target field is determined by the number of digits, plus the places for the plus/minus sign and the decimal separator. `x` The content of the source field is first converted to data type `i` (see above) and then to type `x` (see [conversion table for source field type `i`, `int8`, (`b`, `s`)](ABENCONVERSION_TYPE_IBS.html)). `xstring` The content of the source field is first converted to data type `i` (see above) and then to type `xstring` (see [conversion table for source field type `i`, `int8`, (`b`, `s`)](ABENCONVERSION_TYPE_IBS.html)). `d` The content of the source field is first converted to data type `i` (see above) and then to type `d` (see [conversion table for source field type `i`, `int8`, (`b`, `s`)](ABENCONVERSION_TYPE_IBS.html)). `t` The content of the source field is first converted to data type `i` (see above) and then to type `t` (see [conversion table for source field type `i`, `int8`, (`b`, `s`)](ABENCONVERSION_TYPE_IBS.html)). `utclong` Not supported.
