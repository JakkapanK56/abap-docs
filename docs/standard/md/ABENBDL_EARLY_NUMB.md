\\ `...  early numbering  ...`

Defines unmanaged [early numbering](ABENRAP_EARLY_NUMBERING_GLOSRY.html) for all primary key fields of a [RAP BO entity](ABENRAP_BO_ENTITY_GLOSRY.html). Unmanaged early numbering means:

Unmanaged early numbering is defined in the [entity behavior characteristics](ABENCDS_ENTITY_PROPERTIES_GLOSRY.html) for each [entity behavior definition](ABENCDS_ENTITY_BDEF_GLOSRY.html) separately. That means that some business object entities may use unmanaged early numbering, while others use different methods to fill the primary key fields. The affected key fields must be specified as [`readonly`](ABENBDL_FIELD_CHAR.html).

Alternative options for numbering:

For further conceptual background on early and late numbering, see the development guide for the ABAP RESTful Application Programming Model, section [Numbering](https://help.sap.com/docs/ABAP_Cloud/f055b8bf582d4f34b91da667bc1fcce6/d85aec25222145f0b0cbbe8b02db51f0?version=sap_cross_product_abap).

The following example shows a managed BDEF that defines unmanaged early numbering for the root view entity `DEMO_RAP_EARLY_NUMBERING`. The primary key field `KeyField` is defined as `readonly`.

**Note** This example is intentionally kept short and simple and serves demonstration purposes only.

The ABAP behavior pool implements the local handler class `earlynumbering_create`. This class assigns ascending integer values as primary keys for each entity instance.

-   The [RAP runtime engine](ABENRAP_RUNTIME_ENGINE_GLOSRY.html) assigns values to the primary key fields.
-   How the value assignment takes place is defined by the application developer in the [ABAP behavior pool](ABENBEHAVIOR_POOL_GLOSRY.html) (therefore, it is called *unmanaged*). The implementation must be done in the [RAP handler method](ABENABP_HANDLER_METHOD_GLOSRY.html)\\ [`FOR NUMBERING`](ABAPHANDLER_METH_NUMBERING.html).
-   The key value for an instance is available in the transactional buffer instantly after the [`MODIFY`](ABAPMODIFY_ENTITY_ENTITIES.html) request for `CREATE` (which is why it is called *early* numbering, as opposed to [late numbering](ABENBDL_LATE_NUMBERING.html)).

-   [Managed RAP BOs](ABENMANAGED_RAP_BO_GLOSRY.html)
-   [Unmanaged](ABENUNMANAGED_RAP_BO_GLOSRY.html) and [draft-enabled](ABENDRAFT_RAP_BO_GLOSRY.html) RAP BOs.
-   **Note** Not available in unmanaged RAP BOs without draft handling.
-   In [projection business objects](ABENRAP_PROJECTION_BO_GLOSRY.html) and [RAP BO interfaces](ABENRAP_BO_INTERFACE_GLOSRY.html), early numbering is automatically reused from the [base BDEF](ABENRAP_BASE_BDEF_GLOSRY.html) and does not need to be explicitly defined.

-   [`numbering:managed`](ABENBDL_FIELD_NUMBERING.html)
-   [`late numbering`](ABENBDL_LATE_NUMBERING.html)

-   The ABAP class `CL_DEMO_RAP_MANAGED_EARLY_NUMB` uses [EML](ABENEML_GLOSRY.html) to access to [RAP business object](ABENRAP_BO_GLOSRY.html) and performs the following steps:

-   It creates three entity instances by specifying a value for the field `CharField`.
-   It commits the three new entity instances to the database.
-   Using the ABAP SQL `SELECT` statement, it displays the content of the database table. It can be seen that the primary key field `KEY_FIELD` of each entity instance contains a value, which was generated by means of unmanaged early numbering.
