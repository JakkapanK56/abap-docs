Generally, a table function is a function that returns a tabular result set. Usable in ABAP as follows:

Besides function that return tabular results, there are also [built-in functions](ABENITAB_FUNCTIONS.html) that work with [internal tables](ABENINTERNAL_TABLE_GLOSRY.html) as arguments. Those might also be called table functions.

-   In [ABAP SQL](ABENABAP_SQL_GLOSRY.html), a table function is a function with a tabular result set that can be used as a [data source](ABAPSELECT_DATA_SOURCE.html) in the [`FROM`](ABAPFROM_CLAUSE.html) clause of a [query](ABENASQL_QUERY_GLOSRY.html) and in other operand positions. Currently, ABAP SQL contains only [hierarchy functions](ABENHIERARCHY_FUNCTION_GLOSRY.html) as table functions.
-   In the [SAP HANA database](ABENHANA_DATABASE_GLOSRY.html), a [database function](ABENDATABASE_FUNCTION_GLOSRY.html) with a tabular return value. In [SQLScript](ABENSQL_SCRIPT_GLOSRY.html), table functions can be used as operands in the same places as [database tables](ABENDATABASE_GLOSRY.html) or [views](ABENVIEW_GLOSRY.html). A table function can be managed as an [AMDP function](ABENAMDP_FUNCTION_GLOSRY.html) in an [AMDP class](ABENAMDP_CLASS_GLOSRY.html).
-   In [ABAP CDS](ABENABAP_CDS_GLOSRY.html), [CDS table functions](ABENCDS_TABLE_FUNCTION_GLOSRY.html) are [CDS persistent entities](ABENCDS_SQL_ENTITY_GLOSRY.html) defined using [`DEFINE TABLE FUNCTION`](ABENCDS_F1_DEFINE_TABLE_FUNCTION.html).
