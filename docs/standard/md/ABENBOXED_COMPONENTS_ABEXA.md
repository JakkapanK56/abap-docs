This example demonstrates the memory usage of [boxed components](ABENBOXED_COMPONENT_GLOSRY.html).

Internal tables have a nested structure as a boxed component. When such an internal table has many entries but only a few or none of the boxed components are filled, memory consumption can be significantly reduced compared to an internal table that does not have boxed components.

The example does the following:

-   It defines six internal tables, three of which contain a nested structure as a boxed component, and the other three contain a nested structure that is not a boxed component.
-   In a loop, the tables are populated with many entries under different conditions to ensure that both internal tables, with and without boxed components, have the same content.

-   The components of the nested structures in both internal tables, with and without boxed components, are not populated.
-   All components of the nested structures in both internal tables, with and without boxed components, are populated.
-   Only a few of the components of the nested structures in both internal tables, with and without boxed components, are populated.

-   Once populated, the allocated and used memory for each internal table is retrieved using the `GET_MEMORY_SIZE_OF_OBJECT` method of the `CL_ABAP_MEMORY_UTILITIES` class. The results are then written to another internal table, which is displayed.
-   Notes on the result:

-   Empty nested boxed components vs. empty nested non-boxed components:
-   The internal table with boxed components has considerably less memory allocated than the internal table without boxed components. Although there are no entries in the non-boxed components, there is full memory allocation and usage.
-   All nested boxed components populated vs. all nested non-boxed components populated:
-   The allocated and used memory of the internal table with boxed components is slightly higher than the internal table without boxed components because there are extra administrative memory costs for boxed components.
-   Only a few nested boxed components populated vs. only a few nested non-boxed components populated:
-   In the case of large internal tables with boxed components, where only a few of the boxed components are populated, the allocated and used memory is considerably less compared to the equivalent table without boxed components.
