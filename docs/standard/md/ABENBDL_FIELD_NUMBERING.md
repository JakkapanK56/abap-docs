`... field(numbering:managed) Field1, Field2, ...;`

Defines managed [internal numbering](ABENRAP_INT_NUMBERING_GLOSRY.html) for a [RAP BO field](ABENRAP_BO_FIELD_GLOSRY.html). The field in question is automatically assigned unique values upon creation of a new entity instance. The RAP BO consumer does not have to provide any input for those fields. Managed internal numbering is managed by the [RAP framework](ABENRAP_FRAMEWORK_GLOSRY.html) and no implementation in the ABAP behavior pool is required.

The RAP BO fields `Field1`, `Field2` must meet the following prerequisites:

`Field1`, `Field2`, ... can be defined as [`readonly`](ABENBDL_FIELD_CHAR.html) to ensure that no values are passed from the RAP BO consumer. In this case, it is an internal numbering scenario. However, this is optional. If you do not set the key field to `readonly`, the key value can also be given by the consumer. This is referred to as optional external numbering: The RAP framework only provides key values in case the consumer hands over initial values.

For user-defined numbering, the keyword `early numbering` is available, see the topic [RAP - `early numbering`](ABENBDL_EARLY_NUMB.html).

Development guide for the ABAP RESTful Application Programming Model, section [Automatically Drawing Primary Key Values in Managed BOs](https://help.sap.com/docs/ABAP_Cloud/f055b8bf582d4f34b91da667bc1fcce6/82ce57d6d2014007944a667dcdb830bf?version=sap_cross_product_abap).
