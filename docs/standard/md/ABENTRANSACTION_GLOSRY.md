Execution of an [ABAP program](ABENABAP_PROGRAM_GLOSRY.html) using a [transaction code](ABENTRANSACTION_CODE_GLOSRY.html). There are [dialog](ABENDIALOG_TRANSACTION_GLOSRY.html), [report](ABENREPORT_TRANSACTION_GLOSRY.html), [parameter](ABENPARAMETER_TRANSACTION_GLOSRY.html), [variant](ABENVARIANT_TRANSACTION_GLOSRY.html), and [`OO` transactions](ABENOO_TRANSACTION_GLOSRY.html). A transaction is started by entering the transaction code in the [command field](ABENCOMMAND_FIELD_GLOSRY.html) on the [standard toolbar](ABENSTANDARD_TOOLBAR_GLOSRY.html) or by using the statements [`CALL TRANSACTION`](ABAPCALL_TRANSACTION.html) or [`LEAVE TO TRANSACTION`](ABAPLEAVE_TO_TRANSACTION.html).

At SAP, database transactions are referred to as [database LUWs](ABENDATABASE_LUW_GLOSRY.html).

In a business context, for example in [RAP](ABENRAP_GLOSRY.html), a transaction is refers to the result of a business-related operation that accesses and modifies data. When completed, a consistent state of data is produced.
