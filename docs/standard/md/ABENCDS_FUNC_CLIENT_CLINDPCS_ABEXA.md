This example demonstrates a client-independent CDS table function that is [client-safe](ABENCLIENT_SAFE_GLOSRY.html) and suited for [ABAP for Cloud Development](ABENABAP_FOR_CLOUD_DEV_GLOSRY.html).

This example class accesses the [client-independent](ABENCDS_FUNC_CLIENT_HANDLING.html)\\ [CDS table function](ABENCDS_TABLE_FUNCTION_GLOSRY.html). `DEMO_CDS_TABFUNC_CLIENT_INDPCS`. Client safety is enforced by the annotation `[ClientHandling.clientSafe](ABENCDS_1549616294_ANNO.html)` and the annotation `[ClientHandling.algorithm](ABENCDS_1114072352_ANNO.html)` must be used with the value `#NONE`.

The implementing AMDP class `CL_DEMO_AMDP_TABFNC_CLNTINDPCS` has language version [ABAP for Cloud Development](ABENABAP_FOR_CLOUD_DEV_GLOSRY.html) and the AMDP method `GET_DOKIL_FOR_CDS` is also client-safe by accessing a client-independent CDS view entity `DEMO_CDS_DOKIL` only.

Another CDS table function `DEMO_CDS_TABFUNC_CL_INDPDDCS` does exactly the same, but is implemented by another method of the same AMDP class that accesses a DDIC database table instead of a CDS view entity in a client safe way.
