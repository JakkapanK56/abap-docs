A character-like data object is now sufficient in all operand positions where a [time zone](ABENTIME_ZONE_GLOSRY.html) could previously only be specified with exactly the type `TZNZONE` or as a text field of the length 6. This concerns the statements [`CONVERT [INTO] TIME STAMP`](ABAPCONVERT_TIME-STAMP.html) and formatting of [string templates](ABAPCOMPUTE_STRING_FORMAT_OPTIONS.html) and [`WRITE`](ABAPWRITE_TO_OPTIONS.html).
