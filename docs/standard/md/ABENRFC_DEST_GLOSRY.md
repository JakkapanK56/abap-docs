An RFC destination contains the connection parameters for [remote function calls](ABENREMOTE_FUNCTION_CALL_GLOSRY.html). It contains the connection type, the target system, and the partner program. Connections that only affect another [AS ABAP](ABENAS_ABAP_GLOSRY.html) can be classified as [trusted systems](ABENTRUSTED_SYSTEM_GLOSRY.html). RFC destinations are managed statically using the [transaction](ABENTRANSACTION_GLOSRY.html)\\ `SM59` or dynamically using the class `CL_DYNAMIC_DESTINATION`.
