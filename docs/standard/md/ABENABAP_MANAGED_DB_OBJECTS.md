By default, an [AS ABAP](ABENAS_ABAP_GLOSRY.html) operates with data stored in the [ABAP database schema](ABENABAP_DB_SCHEMA_GLOSRY.html) in its [standard database](ABENSTANDARD_DB_GLOSRY.html). The schema can be accessed using a [standard connection](ABENSTANDARD_DB_CONNECTION_GLOSRY.html). The [database objects](ABENABAP_MANAGED_DB_OBJECT_GLOSRY.html) used to store and access AS ABAP data in ABAP programs, that is, [CDS table entities](ABENCDS_TABLE_ENTITY_GLOSRY.html), [DDIC database tables](ABENDDIC_DB_TABLE_GLOSRY.html), [DDIC database views](ABENDATABASE_VIEW_GLOSRY.html) and [stored procedures](ABENSTORED_PROCEDURE_GLOSRY.html) and so on, are managed by the AS ABAP. The database objects are not created and maintained as native objects in the development environment of the database platform itself. Instead, they are created and maintained using the associated AS ABAP tools and frameworks, such as the [ABAP Dictionary](ABENABAP_DICTIONARY_GLOSRY.html), [ABAP CDS](ABENABAP_CDS_GLOSRY.html), and [AMDP](ABENAMDP_GLOSRY.html).
