An [SQL injection](ABENSQL_INJECTION_GLOSRY.html) is a widely used attack method on insufficiently secure input from outside. An SQL injection passes malicious SQL code injected into a program to the database system, unchecked and unescaped. In ABAP, this can occur when the following programming techniques are used:

-   [Dynamically specified tokens in ABAP SQL](ABENSQL_INJ_DYN_TOKENS_SCRTY.html)
-   [SQL Injections Using Dynamic Tokens in ABAP SQL](ABENSQL_INJ_DYN_TOKENS_SCRTY.html)
-   [SQL Injections Using ADBC](ABENSQL_INJ_ADBC_SCRTY.html)
-   [SQL Injections Using AMDP](ABENSQL_INJ_AMDP_SCRTY.html)
-   [SQL Injections Using Object Services](ABENSQL_INJ_OS_QUERY_SCRTY.html)
-   [SQL Injections Using Generic Programming](ABENSQL_INJ_GEN_PROG_SCRTY.html)
