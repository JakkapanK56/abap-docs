The statements

for creating and editing [ABAP source code](ABENABAP_SOURCE_CODE_GLOSRY.html) dynamically are released for use in application programming. Some additions of the released statements and certain statements are intended for internal system use only and must not be used in application programs.

-   [`GENERATE SUBROUTINE POOL`](ABAPGENERATE_SUBROUTINE_POOL.html),
-   [`READ REPORT`](ABAPREAD_REPORT.html),
-   [`SYNTAX-CHECK`](ABAPSYNTAX-CHECK_FOR_ITAB.html),
-   [`INSERT REPORT`](ABAPINSERT_REPORT.html),

-   The statements described here can be used in application programs in exceptional cases only. ABAP provides many other means of dynamic programming, which generally make creating source code dynamically unnecessary (see the list in [dynamic program processing](ABENABAP_LANGUAGE_DYNAMIC.html)).
-   To display or edit source code in SAP GUI interactively, the class `CL_GUI_SOURCEEDIT` can, for example, be used. This class calls the ABAP Editor control.
