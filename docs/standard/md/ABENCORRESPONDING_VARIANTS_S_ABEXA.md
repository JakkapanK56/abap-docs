This example demonstrates variants of statements with [`MOVE-CORRESPONDING`](ABAPMOVE-CORRESPONDING.html) and the [`CORRESPONDING`](ABENCONSTRUCTOR_EXPR_CORRESPONDING.html) operator using [deep structures](ABENDEEP_STRUCTURE_GLOSRY.html).

Two structures, `struc1` and `struc2`, are declared. Both contain an identically named [nested structure](ABENNESTED_STRUCTURE_GLOSRY.html) and nested table as deep components. The nested structures `struc_nested` differ in terms of the components' data types. The number of components in the nested structures as well as in the two nested internal tables `itab` is different.

The example covers statements with `MOVE-CORRESPONDING` and the `CORRESPONDING` operator demonstrating the differences in the result when using the various additions. The source code contains explanations on those differences.
