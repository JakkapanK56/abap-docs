This example demonstrates parameter passing in two RFC variants.

The function module `DEMO_RFM_PARAMETERS` is called using [synchronous](ABAPCALL_FUNCTION_DESTINATION.html) and [asynchronous](ABAPCALL_FUNCTION_STARTING.html) RFC.

-   In synchronous calls, parameters are passed and applied directly in the statement `CALL FUNCTION`.
-   In asynchronous calls, parameters can only be passed in the statement `CALL FUNCTION`. Parameters are applied using the statement [`RECEIVE`](ABAPRECEIVE.html) in a callback routine. The statement [`WAIT FOR ASYNCHRONOUS TASKS`](ABAPWAIT_ARFC.html) is used to wait for this routine.
