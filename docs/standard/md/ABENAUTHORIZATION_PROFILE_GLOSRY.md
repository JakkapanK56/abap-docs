A summary of multiple [authorizations](ABENAUTHORIZATION_GLOSRY.html). An authorization can be assigned to multiple authorization profiles. Entering authorization profiles in the [user master record](ABENUSER_MASTER_RECORD_GLOSRY.html) assigns [user names](ABENUSER_NAME_GLOSRY.html) to authorizations. Authorization profiles can be generated by the profile generator from role maintenance (transaction `PFCG`) or be displayed using [transaction code](ABENTRANSACTION_CODE_GLOSRY.html)\\ `SU02`.
