When a program is created, the [ABAP language version](ABENABAP_VERSIONS.html)\\ [Standard ABAP](ABENSTANDARD_ABAP_GLOSRY.html) with enabled [Unicode checks](ABENUNICODE_CHECK_GLOSRY.html) is set by default.

In the current [AS ABAP](ABENAS_ABAP_GLOSRY.html) release, only Unicode systems are supported. Programs with the language version [Non-Unicode ABAP](ABENNON_UNICODE_ABAP_GLOSRY.html) can no longer be executed in the current release. The ABAP language version must not be set back to [Non-Unicode ABAP](ABENNON_UNICODE_ABAP_GLOSRY.html). Higher [ABAP language versions](ABENABAP_VERSION_GLOSRY.html) are possible because they cover the Unicode checks.

[Program Properties](ABENPROGRAM_PROPERTY_GUIDL.html)

-   Programs with the language version [Standard ABAP](ABENSTANDARD_ABAP_GLOSRY.html) work in [Unicode systems](ABENUNICODE_SYSTEM_GLOSRY.html) and non-Unicode systems.
-   Programs with the [obsolete](ABENABAP_VERSIONS_OBSOLETE.html) language version [Non-Unicode ABAP](ABENNON_UNICODE_ABAP_GLOSRY.html) only work in non-Unicode systems.
