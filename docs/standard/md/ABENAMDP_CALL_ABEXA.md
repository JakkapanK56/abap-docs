This example demonstrates how a database procedure is called from an AMDP procedure. The AMDP method is not read only and not [client-safe](ABENCLIENT_SAFE_GLOSRY.html).

The [SQLScript procedure](ABENSQL_SCRIPT_PROCEDURE_GLOSRY.html) of the AMDP method `INCREASE_PRICE` in the AMDP class `CL_DEMO_AMDP_CALL` calls a database procedure that is not managed by AMDP of the current database schema in the namespace `/1BCAMDP/`:

The called database procedure is created dynamically in the static constructor of the AMDP class `CL_DEMO_AMDP_CALL` using [ADBC](ABENADBC.html). It is in the same database schema, but due to the namespace `/1BCAMDP/` it cannot and must not be used after the addition [`USING`](ABAPMETHOD_BY_DB_PROC.html) of the `METHOD` statement. Compare this [executable example](ABENAMDP_CALL_AMDP_ABEXA.html) that calls another AMDP procedure.
