This example demonstrates an [ABAP Daemon](ABENABAP_DAEMON_GLOSRY.html).

The class demonstrates the most important aspects of an ABAP daemon and consists of the class above and two further classes. The repository object of the daemon has the property *Explicitly Started Daemon (same user)*.

This class makes it possible to start an ABAP daemon interactively and then perform various actions using the daemon. The checkboxes have the following meaning:

The class creates an output that displays the individual actions in both the class and in the ABAP daemon. To do this, the daemon writes entries to the [export/import table](ABENEXPORT_IMPORT_TABLE_GLOSRY.html)\\ `DEMO_INDX_BLOB`, which are then read by the class after a short wait time. If the current system is slow, this wait time may not be long enough to collect all actions of the daemon. The ABAP daemon on the current AS instance can be monitored in parallel to the execution of the class using the transaction `SMDAEMON`.

The class `CL_DEMO_ABAP_DAEMON_DEMO` does not work directly with the class `CL_ABAP_DAEMON_CLIENT_MANAGER` and any reads performed on this class are wrapped in the class `CL_DEMO_ABAP_DAEMON_BROKER` instead. This is because `CL_ABAP_DAEMON_CLIENT_MANAGER` can only be used to access an ABAP daemon in the program in which the daemon was started. It must be possible for the daemon itself to also create an instance of its class in this example, which means the reads must be moved to a program that can be accessed by the ABAP program and the ABAP daemon. To enable this, `CL_DEMO_ABAP_DAEMON_BROKER` contains the following methods:

The name of the ABAP daemon and the ABAP daemon class are defined as constants of the class `CL_DEMO_ABAP_DAEMON_BROKER`.

The class `CL_DEMO_ABAP_DAEMON` is a subclass of `CL_ABAP_DAEMON_EXT_BASE` and the ABAP daemon class for this example. It implements the most important methods of the interface `IF_ABAP_DAEMON_EXTENSION` and further standalone helper methods. It also implements the interface `IF_ABAP_TIMER_HANDLER` so that it can be an [ABAP Timer handler](ABENABAP_TIMER_HANDLER_GLOSRY.html) for an [ABAP Timer](ABENABAP_TIMER_GLOSRY.html). Although daemons should never be stopped, daemons created by this example should be deleted automatically after an hour, if it is not stopped explicitly first.

In this example, the remaining methods of the interface `IF_ABAP_DAEMON_EXTENSION` only write log entries.

This simple example does not guarantee that an ABAP daemon of the ABAP daemon class `CL_DEMO_ABAP_DAEMON` is a system-wide singleton. Due to parallel accesses that cause restarts, multiple unwanted daemons may occur unintentionally. This applies in particular during the relocation of daemons to another AS instance. Much more effort is required to create a real singleton. See the class `CL_AD_EXT_SIMPLE_DAEMON`, which can be used by the program `RS_ABAP_DAEMON_TEST`.

-   `START_DAEMON`
-   Attempt to start an ABAP daemon. Only one ABAP daemon of the used ABAP daemon class may exist. When the class is executed, a version number is passed in [PCP](ABENPCP_GLOSRY.html) format. If an ABAP daemon already exists, this daemon is used.
-   `GET_INFORMATION`
-   Gets information about the ABAP daemon and displays it.
-   `SEND_MESSAGE`
-   Sends a text message to the ABAP daemon in [PCP](ABENPCP_GLOSRY.html) format.
-   `TRIGGER_AMC`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon instructing the daemon to send an [AMC](ABENAMC_GLOSRY.html) message. The execution waits until the message arrives at a dedicated AMC receiver.
-   `TRIGGER_EXCEPTION`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon, which raises an exception there. The exception produces a [short dump](ABENSHORT_DUMP_GLOSRY.html) that can be viewed in transaction `ST22` and the ABAP daemon is restarted automatically. This raises the version number by 1.
-   `TRIGGER_BLOCKING`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon instructing it to create a statement forbidden in [non-blocking mode](ABENADF_NON_BLOCKING_MODEL.html). This raises an exception and [short dump](ABENSHORT_DUMP_GLOSRY.html), which can be viewed in transaction `ST22`. The ABAP daemon is then restarted automatically. This raises the version number by 1.
-   `TRIGGER_RESTART`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon that restarts the daemon. This raises the version number by 1.
-   `TRIGGER_RELOCATION`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon instructing the daemon to create a new instance of its class on a different AS instance and delete the previous instance.
-   `TRIGGER_STOP`
-   Sends a [PCP](ABENPCP_GLOSRY.html) message to the ABAP daemon instructing it to stop.
-   `STOP_DAEMON`
-   Stops the daemon.
