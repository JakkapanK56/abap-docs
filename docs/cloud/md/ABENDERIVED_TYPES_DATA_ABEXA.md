This example demonstrates the use of `%data` and `%target` with a [managed](ABENBDL_IMPL_TYPE.html)\\ [RAP BO](ABENRAP_BO_GLOSRY.html).

**Data model**

The CDS data model consists of the root entity `DEMO_MANAGED_ROOT_4` and its child entity `DEMO_MANAGED_CHILD_4`.

**Behavior implementation**

For the above RAP behavior definition, one [ABP](ABENBEHAVIOR_POOL_GLOSRY.html) is created. The global class of the behavior pool is `BP_DEMO_MANAGED_ROOT_4`. The actual behavior implementation takes place in local classes that are defined and implemented in the `BP_DEMO_MANAGED_ROOT_4========CCIMP` of the behavior pool. In this case, the behavior pool does not include any implementations that are relevant for the example.

**Access with ABAP using EML**

There is one ABAP EML modify request that covers several RAP modify operations:

With a [`COMMIT ENTITIES`](ABAPCOMMIT_ENTITIES.html) statement, the instances are saved to the database. The output shows the database table entries of the root and child entity.

1.  The first ABAP EML modify operation is a RAP create operation that creates two [RAP BO instances](ABENRAP_BO_INSTANCE_GLOSRY.html) for the root entity. The first instance is created by specifying the key and data fields with `%data` individually with the [structure component selector](ABENSTRUCTURE_COMPONENT_SEL_GLOSRY.html). The second instance is created by specifying `%data` with the [`VALUE`](ABENCONSTRUCTOR_EXPRESSION_VALUE.html). In principle, the use of `%data` is optional. For example, the data fields can be specified directly without `%data`.
2.  The second ABAP EML modify operation is a create-by-association focusing on the `%target` component. Two RAP BO instances are created for the child entity. The reference to instances from the root entity is made via [`%cid_ref`](ABAPDERIVED_TYPES_CID_REF.html). Also here, the key and data fields of the instance that is to be created for the child entity within `%target` are specified using `%data` which is optional in this case, too.
3.  The third ABAP EML modify request is an update operation. It just shows the optional use of `%data` when referring to key and data fields.
