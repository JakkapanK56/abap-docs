Superordinate term for [assertions](ABENASSERTION_GLOSRY.html), [breakpoints](ABENBREAKPOINT_GLOSRY.html), or [logpoints](ABENLOGPOINT_GLOSRY.html). The statements [`ASSERT`](ABAPASSERT.html), `BREAK-POINT`, and `LOG-POINT` for defining checkpoints are not operative but equip the program for test purposes. Inactive checkpoints are ignored during program execution.
