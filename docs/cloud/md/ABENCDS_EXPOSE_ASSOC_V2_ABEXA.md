This example demonstrates how CDS associations are [exposed](ABENCDS_SELECT_LIST_ASSOCIATION_V2.html) in the `SELECT` list of a CDS view entity.

The executable example shown above demonstrates how [CDS associations](ABENCDS_ASSOCIATION_GLOSRY.html) are accessed that are exposed in the `SELECT` list of a CDS view entity. For each view, the data source in the `SELECT` statement is specified not directly, but as path expression, making use of an exposed association. For comparison, the explicit join expression that produces the same result is shown.

-   The following CDS view entity exposes its CDS association `_demo_join2`, which associates its data source `demo_join1` with the DDIC database table `demo_join1`. The field `d` used in the `ON` condition of the [association source](ABENASSOCIATION_SOURCE_GLOSRY.html) must also be an element of the `SELECT` list.

-   The following CDS view entity matches the preceding view, but has an additional `WHERE` condition.

-   The following CDS view entity accesses the view `demo_cds_expose_assoc_1_ve` above. It does not have its own CDS association, but it exposes the CDS association of its data source `demo_cds_expose_assoc_1_ve._demo_join2`. Here, the field `demo_cds_expose_assoc_1_ve.d` of the [association source](ABENASSOCIATION_SOURCE_GLOSRY.html) used in the `ON` condition must again be part of the `SELECT` list.

-   The following CDS view entity matches the preceding view, but has an additional `WHERE` condition.

-   The following CDS view entity accesses the view `demo_cds_expose_assoc_1a_ve` above. It does not have its own CDS association, but it exposes the CDS association of its data source `demo_cds_expose_assoc_1a_ve._demo_join2`.

-   The following CDS view entity matches the preceding view, but has an additional `WHERE` condition.
