RAP behavior definitions must be explicitly enabled to allow for behavior extensibility. They must fulfill technical requirements and define extension points. Otherwise, no BDEF extensions are possible. The following sections describe how to enable RAP behavior definitions of type [managed](ABENMANAGED_RAP_BO_GLOSRY.html), [unmanaged](ABENUNMANAGED_RAP_BO_GLOSRY.html), [interface](ABENRAP_BO_INTERFACE_GLOSRY.html), [projection](ABENCDS_PROJ_BDEF_GLOSRY.html), and [abstract](ABENCDS_ABSTRACT_BDEF_GLOSRY.html) for behavior extensibility. It also describes how to enable a BDEF extension for extensibility to allow for [extension layering](ABENRAP_EXT_LAYERING_GLOSRY.html).

Development guide for the ABAP RESTful Application Programming Model, section [*Extensibility-Enablement for Behavior Extensions*](https://help.sap.com/docs/ABAP_Cloud/f055b8bf582d4f34b91da667bc1fcce6/b502fdb6fa864d14ac5960a90927ed1f?version=sap_cross_product_abap).

For field and node extensibility, the RAP data model must be enabled for extensibility. This is described in the following sections:

-   [RAP - Extensibility Enabling for Base BOs](ABENBDL_EXTENSIBILITY_ENABLING_M_U.html)
-   [RAP - Extensibility Enabling for RAP BO Interfaces](ABENBDL_EXTENSIBILITY_ENABLNG_INTF.html)
-   [RAP - Extensibility Enabling for Projection BDEFs](ABENBDL_EXT_ENABLING_PROJECTION.html)
-   [RAP - Extensibility Enabling for Abstract BDEFs](ABENBDL_EXT_ENABLING_ABSTRACT.html)
-   [RAP - Extensibility Enabling for BDEF Extensions](ABENBDL_EXTENSIBILITY_ENABLING_EXT.html)
