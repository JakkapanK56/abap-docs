This example compares the generation of a [level-based hierarchy](ABENLEVEL_BASED_HIERA_GLOSRY.html) to general grouping.

The example shows that the generation of a [level-based hierarchy](ABENLEVEL_BASED_HIERA_GLOSRY.html) can be explained by the grouping of data:

This example is for demonstration purposes only. The results of the grouping are no SQL hierarchies and cannot be used as such. There are no additional hierarchy columns and they cannot be used in [hierarchy navigators](ABENSELECT_HIERARCHY_NAVIGATORS.html). The performance of the hierarchy generator based on a built-in HANA function is better by far.

-   By using the addition [`GROUP BY`](ABAPLOOP_AT_ITAB_GROUP_BY.html) of the `LOOP` statement for the data source `level_tab`, the same result can be produced as by the hierarchy generator.
-   By using the [`GROUP BY`](ABAPGROUPBY_CLAUSE.html) clause of the `SELECT` statement for the data source `level_tab`, the same result can be produced as by the hierarchy generator.
