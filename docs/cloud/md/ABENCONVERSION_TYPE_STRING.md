The class [`CL_ABAP_DECFLOAT`](ABENCL_ABAP_DECFLOAT_DOC.html) contains the methods `READ_DECFLOAT34` and `READ_DECFLOAT16` for converting character strings into decimal floating point numbers. The exceptions of these methods are more significant than those of a regular assignment. Furthermore, the methods return a return value that reveals information about the rounding that was performed.

The result of the following conversion has the value *1123*.

The string resulting from the assignment is *abcd\\ \\ \\ \\* with length 8. The trailing blanks of the literal with type `string` and length 8 are respected.

The byte chain resulting from the conversion is `FFFF0000`.

The value of the date field resulting from the conversion is `00000000`. If, on the other hand, a text field of type `c` containing only blanks is converted, this produces a target field containing eight blanks, despite the cutting off of all trailing blanks.

Conversion of a text string composed of date and time specifications in ISO format 8601 to a value in a [time stamp field](ABENTIMESTAMP_FIELD_GLOSRY.html)\\ `ts_conv`. The value in `ts_conv` is a valid [UTC time stamp](ABENUTC_TIMESTAMP_GLOSRY.html) but not the real UTC time stamp that belongs to the system date and system time in the system time zone. For the latter, the statement [`CONVERT INTO UTCLONG`](ABAPCONVERT_DATE_UTCLONG.html) can be used that performs a real time stamp conversion instead of a simple concatenation.

-   In ABAP, the [trailing blanks](ABENABAP_DATA_STRING.html) are respected for source fields of type `string` but not for data objects of type `c`.
