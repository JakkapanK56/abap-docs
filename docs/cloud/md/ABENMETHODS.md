Methods are declared using the statements

These are only possible in the declaration part of classes and interfaces.

Methods determine the behavior of a class. [Instance methods](ABENINSTANCE_METHOD_GLOSRY.html) are declared by the statement [`METHODS`](ABAPMETHODS.html), [static methods](ABENSTATIC_METHOD_GLOSRY.html) are declared by the statement [`CLASS-METHODS`](ABAPCLASS-METHODS.html). The declaration defines the interface of a method. For special tasks there are different types of methods:

The following methods are also available:

-   [`METHODS`](ABAPMETHODS.html) for instance methods
-   [`CLASS-METHODS`](ABAPCLASS-METHODS.html) for static methods

-   General [instance methods](ABAPMETHODS_GENERAL.html) and [static methods](ABAPCLASS-METHODS_GENERAL.html)
-   Functional [instance methods](ABAPMETHODS_FUNCTIONAL.html) and [static methods](ABAPCLASS-METHODS_FUNCTIONAL.html)
-   [Instance constructors](ABAPMETHODS_CONSTRUCTOR.html) and [static constructors](ABAPCLASS-METHODS_CONSTRUCTOR.html)
-   [Event handlers](ABAPMETHODS_EVENT_HANDLER.html) and [static event handlers](ABAPCLASS-METHODS_EVENT_HANDLER.html)

-   [Test methods](ABAPMETHODS_TESTING.html) of ABAP Unit, which can only be declared as instance methods.
-   [AMDP](ABENAMDP_METHODS.html) methods, which are implemented in a database-specific language and not in ABAP.
