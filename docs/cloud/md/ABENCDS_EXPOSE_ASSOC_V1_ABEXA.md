This example demonstrates how CDS associations are [exposed](ABENCDS_SELECT_LIST_ASSOCIATION_V1.html) in the `SELECT` list of a CDS view.

The example demonstrates how [CDS associations](ABENCDS_ASSOCIATION_GLOSRY.html) are accessed that are exposed in the `SELECT` list of CDS views. For each view, the data source is specified not directly, but as path expression, making use of an exposed association. As a comparison, the explicit join expression that produces the same result is shown.

-   The following CDS view exposes its CDS association `_demo_join2`, which associates its data source `demo_join1` with the DDIC database table `demo_join1`. The field `d` used in the `ON` condition of the [association source](ABENASSOCIATION_SOURCE_GLOSRY.html) must also be an element of the `SELECT` list.

-   The following CDS view matches the preceding view, but has an additional `WHERE` condition.

-   The following CDS view accesses the view `demo_cds_expose_assoc_1` above. It does not have its own CDS association, but it does expose the CDS association of its data source `demo_cds_expose_assoc_1._demo_join2`. Here, the field `demo_cds_expose_assoc_1.d` of the [association source](ABENASSOCIATION_SOURCE_GLOSRY.html) used in the `ON` condition must again be part of the `SELECT` list.

-   The following CDS view matches the preceding view, but has an additional `WHERE` condition.

-   The following CDS view accesses the view `demo_cds_expose_assoc_1a` above. It does not have its own CDS association, but it does expose the CDS association of its data source `demo_cds_expose_assoc_1a._demo_join2`.

-   The following CDS view matches the preceding view, but has an additional `WHERE` condition.
