This example demonstrates the deserialization of empty elements.

An elementary data object, a structure and an internal table are transformed using the [identity transformation](ABENID_TRAFO_GLOSRY.html)\\ `ID` to their [asXML](ABENASXML_GLOSRY.html) and [asJSON](ABENASJSON_GLOSRY.html) representations. In the asXML representation, the values of all elements are removed. In the asJSON representation, only the components from objects and arrays can be removed.

After deserialization to the original ABAP data objects, the elementary data object and the internal table are initial. However, the empty element in the structure is interpreted as a missing component and the structure component retains its previous value. If the transformation option [`clear`](ABAPCALL_TRANSFORMATION_OPTIONS.html) is used with the value *all*, all ABAP data objects are initialized.

The modification of XML and JSON data using string processing shown here is only for the sake of clarity in the example. In production programs, the APIs of [class libraries for XML](ABENABAP_XML_LIBS.html) should be used for this.
