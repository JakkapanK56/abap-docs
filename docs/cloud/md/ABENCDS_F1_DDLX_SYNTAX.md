Alongside the [DDL statements for data definitions](ABENCDS_F1_DDL_SYNTAX.html), the [CDS DDL](ABENCDS_DDL_GLOSRY.html) in ABAP CDS makes it possible to define [metadata extensions](ABENCDS_META_DATA_EXTENSIONS.html) for CDS entities in [`DDLX` source code](ABENDDLX_SOURCE_CODE_GLOSRY.html). The following statement is used here:

-   [`ANNOTATE ENTITY, VIEW`](ABENCDS_F1_ANNOTATE_VIEW.html)

-   Two slashes (`//`) can be used as [comment characters](ABENCDS_GENERAL_SYNTAX_RULES.html) in [`DDLX` source code](ABENDDLX_SOURCE_CODE_GLOSRY.html) for metadata extensions, but not two dashes (`--`).
-   The [`DDLX` source code](ABENDDLX_SOURCE_CODE_GLOSRY.html) of metadata extensions has its own editor in [ADT](ABENADT_GLOSRY.html), which is documented there.
