``\  ...INCLUDE `aspect`.[`element1`](ABENCDS_ASPECT_ELEMENT.html)[,     INCLUDE `aspect`.[`element2`](ABENCDS_ASPECT_ELEMENT.html)],     ...  |\ \{INCLUDE `aspect`.*[excluding       [`element1`](ABENCDS_ASPECT_ELEMENT.html),       [`element2`](ABENCDS_ASPECT_ELEMENT.html),       ...]\}``

Includes 1 to n elements of a CDS aspect in the data definition of a [CDS view entity](ABENCDS_V2_VIEWS.html) or a [CDS projection view](ABENCDS_PROJ_VIEWS.html). The syntax for including elements of an aspect is the aspect and element names, separated by a `.`, such as `include.element`. The same element of an aspect can be included multiple times as long as the alias name of the element is unique.

To include all elements of an aspect, the syntax `include.*` is used. If `include.*` is used, an `excluding \{ ... \}` clause can be added to exclude certain elements of the CDS aspect.

The following example showcases how a CDS aspect is bound to a CDS view entity, as well as how some or all of these bound fields can be included in the entity.
