The new statement [`mapping for`](ABENBDL_TYPE_MAPPING.html) can be used to map field names from legacy code to field names from the current data model. Available for unmanaged and managed RAP BOs.

The new statement [`with additional save`](ABENBDL_SAVING.html) can be used to enhance the default save sequence in a managed RAP BO.

The new statement [`with unmanaged save`](ABENBDL_SAVING.html) can be used to implement an own saving strategy in a managed RAP BO.

The new statement [`group`](ABENBDL_GROUPING.html) can be used to divide the implementation-relevant parts of a BO's business logic into several groups for behavior implementation.
