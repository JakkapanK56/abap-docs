This example demonstrates the deserialization of empty elements.

An elementary data object, a structure and an internal table are transformed using the [identity transformation](ABENID_TRAFO_GLOSRY.html)\\ `ID` to their [asXML](ABENASXML_GLOSRY.html) and [asJSON](ABENASJSON_GLOSRY.html) representations. In the asXML and asJSON representations, all elements or object components are removed.

In a standard deserialization to the original ABAP data objects, the objects retain their original values. If the transformation option [`clear`](ABAPCALL_TRANSFORMATION_OPTIONS.html) is used with the value *all*, all ABAP data objects are initialized.

The modification of XML and JSON data using string processing shown here is only for the sake of clarity in the example. In production programs, the APIs of [class libraries for XML](ABENABAP_XML_LIBS.html) should be used for this.
